# Monthly Maintenance Checklist

Run this checklist about once per month.

## Public Entry Points

- Check GitHub Pages loads: `https://onsenojisan.github.io/omega-library/`
- Check README and `index.md` still describe the same routing roles.
- Check top CTAs still point to the intended targets.

## Canonical And Reference Links

- Check Canonical Overview link.
- Check Reference Guide link.
- Check Reproducibility Package link.
- Confirm Zenodo remains the canonical public record layer.

## Execution Links

- Check the 30 sec Colab link.
- Check the custom CSV template Colab link.
- Confirm custom CSV links do not point to a fixed-domain notebook.

## Issue Routing

- Confirm executable reproduction results, notebook/Colab issues, CSV template issues, and result discussions route to `omega-repro`.
- Confirm navigation, broken links, artifact placement, documentation structure, and Ω Library architecture issues route to `omega-library`.

## Markdown Readability

- Check mobile readability of GitHub Reporting links.
- Check that link lists are not visually merged or overly dense.

## Over-Linking Control

- Avoid adding multiple entry paths to the same artifact unless necessary.
- Preserve clear separation between canonical, reference, executable, and navigation roles.

## Artifact Placement

- Review whether any new artifact needs placement in the Ω Library navigation.
- Do not add new artifacts to front-page navigation unless their role is clear.
