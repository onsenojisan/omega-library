# Agent Instructions

> Maintenance note: This file contains repository operation guidance for AI/code agents. It is public for transparency, but it is not the ordinary reader path. Ordinary readers should start with `README.md` or `index.md`.

## 1. Repository Role

This repository is the public navigation and execution hub for Omega / Ω structural concentration tests.

It is not the source of new theoretical definitions, new empirical domains, or modified evaluation semantics. Preserve the role of `omega-library` as the public entry point that routes readers to executable resources, canonical records, and reporting paths.

Do not reframe the repository, pages, links, maintenance notes, or issue-routing language as prediction, trading performance, causality, intervention, optimization, or endorsement.

## 2. Fixed Omega Minimal Protocol

- Preserve consistency with the fixed Omega / Ω minimal protocol.
- Do not change Omega definitions, high Omega thresholds, event definitions, evaluation outputs, or domain specifications unless explicitly requested.
- Preserve the independence between Omega / high Omega definitions and event definitions.
- Keep evaluation language aligned with:
  - `P(event | high Omega)`
  - baseline `P(event)`
  - ratio
  - counts when available
- Treat null results as valid outcomes.
- Do not introduce post hoc threshold optimization, parameter tuning to improve results, or performance-selection framing.

## 3. Public Navigation Stability

- Preserve stable navigation paths to reproducibility packages and execution resources.
- Preserve Colab entry points.
- Preserve GitHub reporting paths.
- Preserve Zenodo canonical and reference records where already used.
- Avoid excessive cross-linking.
- Do not add links merely to increase density or silence validator warnings.
- Prefer minimal public navigation over broad link expansion.
- Keep the public entry path simple: run, reproduce, use your own data, report results or issues.

## 4. Change Discipline

- Do not edit public-facing files unless the user explicitly requests that scope.
- Do not modify `README.md`, `index.md`, docs, links, workflows, maintenance records, or public page files unless explicitly requested.
- Prefer review notes before public-facing edits.
- Prefer minimal edits over broad rewriting.
- If a requested change conflicts with the existing public structure or fixed Omega minimal protocol, report the conflict before making broader edits.
- Explain any public-facing impact before modifying public-facing files.

## 5. Agent Workflow

- First inspect `git status --short` and recent commits.
- Classify findings as:
  - required fix
  - optional improvement
  - no action
- When checking public-facing material, verify terminology, links, Markdown table integrity, Colab links, GitHub issue/reporting paths, and Zenodo references when relevant.
- Report the final diff and verification commands after edits.
- Do not commit or push unless explicitly requested.

## 6. Prohibited Actions

- Changing fixed Omega / Ω definitions without explicit request.
- Changing high Omega thresholds without explicit request.
- Changing event definitions, evaluation outputs, or domain specifications without explicit request.
- Prediction framing.
- Trading-performance framing.
- Causal or intervention claims.
- Optimization or endorsement framing.
- Excessive cross-linking or link-density edits.
- Adding links only to silence validator warnings.
- Large public navigation rewrites without prior conflict review.
